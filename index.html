API

API

The package surface is intentionally compact: build specs, create stories, and register idioms when you need to extend the grammar.

sl.bar("./data.csv") — data URL inline. Chain .x(), .y(), .where(), .key() to build a state.
sl.seq() orders states into a navigable sequence. .add(state, text) appends; .next() / .goto(n) navigate.
sl.chart(seq, { target, d3, aq }) mounts a standalone animated chart. Returns { to(n), resize, destroy }.
seq.bind({ chart, text }) wires chart + caption element. After binding, seq.next() drives both in one call. 
import * as sl from "scrollylite";

// 1. Build states — data URL inline, no registration
const base = sl.bar("./data.csv")
  .x("decade").y("count").key("decade");

// 2. Sequence — ordered states with narrative text
const sq = sl.seq()
  .add(base,                          "All decades")
  .add(base.where({ type: "Hot days" }), "Filter to hot")
  .add(base.breakdown("type"),          "Split by type");

// 3. Mount + bind — seq.next() drives chart & caption together
const chart = await sl.chart(sq, { target: "#vis", d3, aq });
sq.bind({ chart, text: "#caption" });
btn.onclick = () => sq.next();   // one line drives everything

Task map
  1. Use ScrollyLite when the user wants a narrated scrolling data story.
  2. Build states with seq().add(state, text); avoid hand-writing animation code.
  3. Choose one built-in idiom: bar, line, point, or unit.
  4. Use import * as sl from "scrollylite"; call sl.render() for the full story or sl.chart() for a standalone embed.
  5. Use .key() for stable semantic identity across transitions.

Control modes

Wire any input to any chart.

chart.to(n) is the single public API for driving transitions. Wire it to a button, slider, scroll offset, keyboard shortcut, or timer — the same call, the same animated transition.

Discrete

chart.to(n)

Snap to any step index from any event — button click, keyboard shortcut, route change, or timer. 

const base = sl.bar("./data.csv")
  .x("decade").y("count").key("decade");

const chart = await sl.chart(
  sl.story()
    .step("Hot days",  base.where({ type: "Hot days" }))
    .step("Breakdown", base.breakdown("type"))
    .toSpec(),
  { target: "#chart", d3, aq }
);

// Button click → snap to next step
let n = 0;
btn.onclick = () => chart.to(n ^= 1);
Hot days Breakdown
Slider

chart.to(n) from any input

Wire chart.to(n) to a range input, scroll offset, or any continuous value — same API, same animated transitions. 

// Same chart, same states — driven by a slider
const STEPS = [
  base.where({ type: "Hot days" }),
  base.where({ type: "Hot days", period: "recent" }),
  base.breakdown("type"),
];

// slider min=0, max=STEPS.length-1, step=1
slider.oninput = () =>
  chart.to(+slider.value);
Hot days step 0

Why ScrollyLite

Built for concise, inspectable stories.

No framework. No build step required for static pages. Just D3, Arquero, and a declarative spec.

🎯

State-first authoring

Describe each step as a chart state. ScrollyLite classifies and animates the transitions automatically.

📊

Four built-in idioms

Bar, line, point, and unit charts share the same encoding, filtering, keying, and tooltip vocabulary.

🔍

Semantic scene inference

Changes are classified as focus, guide, granularity, or observation so transitions stay meaningful.

🎨

Themeable

CSS token system with three themes (Default, Arch, Paper). Swap the stylesheet; every chart updates.

🔌

Extensible idioms

Register custom chart idioms with the same plugin system used by the built-ins.

🤖

Agent readable

Compact grammar and an llms.txt reference make spec generation reliable with code-generation AI.
