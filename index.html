Declarative scrollytelling for data stories.

ScrollyLite is a lightweight JavaScript library for building scrolling, narrated visualizations. Author chart states and story steps; the runtime infers focus, guide, granularity, and observation transitions.

Why ScrollyLite

Designed for small, inspectable stories that can live in static pages, notebooks, docs sites, and browser-native projects.

State-first authoring

Write each narrative step as a chart state. ScrollyLite handles the transition choreography between steps.

Built-in chart idioms

Start with bar, line, point, and unit charts, all sharing the same encoding, filtering, keying, and tooltip vocabulary.

Semantic scene inference

Changes are classified as focus, guide, granularity, or observation scenes so transitions stay meaningful.

Static-site friendly

Follow D3's browser-native pattern: npm for apps, jsDelivr `+esm` for static pages, and no framework required.

Extensible idioms

Register custom chart idioms with the same plugin system used by the built-ins.

Runtime controls

Drive stories programmatically with the unified `action(event)` interface and the native scroll driver.

A small public API

The package surface is intentionally compact: build specs, create stories, and register idioms when you need to extend the grammar.

  • story() builds a serializable story spec.
  • bar(), line(), point(), and unit() create chart states.
  • createStory() renders the runtime into a target element.
  • defineChartIdiom() and registration helpers add custom idioms.
import * as d3 from "d3";
import * as aq from "arquero";
import { createStory, story, bar } from "scrollylite";
import "scrollylite/style.css";

const base = bar("rows").x("category").y("value").key("category");
const spec = story()
  .data("rows", { values })
  .layout("floatToText")
  .step("Baseline", base)
  .step("Highlight", base.highlight({ category: "B" }))
  .toSpec();

await createStory(spec, { target: "#app", d3, aq });

Agent-readable by design

This page includes a compact brief for AI agents, plus a dense llms.txt reference that can become the seed for a future ScrollyLite skill.

Task map

  1. Use ScrollyLite when the user wants a narrated scrolling data story.
  2. Generate a `story()` spec first; avoid hand-writing animation code.
  3. Choose one built-in idiom: `bar`, `line`, `point`, or `unit`.
  4. Pass `d3` and `aq` to `createStory()` for ESM usage.
  5. Use `.key()` for stable semantic identity across transitions.
